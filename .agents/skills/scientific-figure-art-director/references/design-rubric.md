# Scientific figure design rubric

A figure is publication-ready only if it passes all core criteria below.

## 1. Message and visual hierarchy

- The main scientific message is obvious without reading source code.
- The strongest visual element corresponds to the main comparison or trend.
- Supporting elements such as gridlines, backgrounds, and strips are quiet.
- The plot does not waste space on blank panels, redundant legends, or decorative ink.

## 2. Geometry matches the estimand

- Estimates with uncertainty use point-range, linerange, ribbon, or interval geoms as appropriate.
- Continuous predictor effects use lines and ribbons, not disconnected bars.
- Discrete contrasts use points and intervals, not misleading connected lines unless ordering is meaningful.
- Correlations use a reference line at zero and a scale that respects the `[-1, 1]` domain.

## 3. Uncertainty is honest

- Intervals are drawn only when lower and upper bounds are finite and provenance supports them.
- Rows without intervals remain visible as point or line estimates.
- Interval type is described in labels, caption, or documentation when ambiguity matters.
- No fake standard errors, caps, ribbons, or alpha bands are invented from unsupported data.

## 4. Scales are interpretable

- Axes have informative names and units or scale labels.
- Limits are chosen for interpretation, not merely for default expansion.
- Parameters with incompatible units or orders of magnitude are not compared on a shared y-axis.
- Transformations are labelled and justified.

## 5. Facets and layout

- Facets are used only when they reduce cognitive load.
- Each facet has enough content to justify its existence.
- Strip labels are short, readable, and meaningful.
- Long parameter names are wrapped and aligned.
- The aspect ratio suits the data: forest plots need height; prediction surfaces need enough width to show trends.

## 6. Typography and labels

- Text is large enough for a manuscript figure after reduction.
- Axis labels use scientific language rather than raw column names.
- Tick labels are not crowded or over-precise.
- Mathematical/statistical notation is correct and consistent.

## 7. Color and accessibility

- Palette is colorblind-safe and print-safe.
- Important distinctions are also encoded by position, line type, shape, labels, or facets.
- Color is used sparingly and consistently across package functions.
- Grey is used deliberately for support elements, not as the dominant default background.

## 8. Export readiness

- Vector PDF or SVG is available for line art.
- Raster output, if needed, is at least 300 dpi for color/grayscale and 600 dpi for line-heavy figures.
- Margins prevent clipping of labels and annotations.
- The figure is legible at common manuscript widths: about 85 mm single column and 170-180 mm double column.

## 9. Reproducibility and package integration

- The plot is generated by a documented package function.
- The function returns a ggplot object.
- Examples run quickly and are robust to optional dependencies.
- Visual snapshots or structural plot tests cover important behavior.

## 10. Failure conditions

Reject the figure if any of these are true:

- It looks like unmodified ggplot2 defaults.
- It has cramped, mostly empty, or misleading facets.
- Labels are unreadable at manuscript size.
- The main scientific comparison is unclear.
- Intervals are misleading, unsupported, or silently dropped.
- Multiple parameters with incompatible scales are forced into one visual comparison.
- Legends repeat information already in facet strips or axis labels.
