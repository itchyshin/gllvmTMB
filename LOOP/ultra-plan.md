# Frozen ultra-plan — gllvmTMB 0.6 five-macro-arc programme

**Approved direction:** one solo Codex/Claude-sequential platform lane; EVA is
intended for 0.6, attempted universally only cell-by-cell after narrow q1 proof,
with automatic scale-back and a Laplace-only fallback. This file records the
binding execution model; later reality may send the plan back to a human gate,
but the loop may not silently rewrite it.

## Reconciliation of the estimator decisions

The statement “EVA is in 0.6” is a scope intention, not a scientific admission.
Design 85 is a landed NO-GO for the high-dimensional non-Gaussian VA campaign:
its pilot mixed the fixed-rank Gate-3 experiment with ML-rank handoff, left
optimiser failures, and never admitted q4/q6 or larger campaigns. Those data
cannot be relabelled or tuned into a GO.

The admissible reset is a new Design 86: complete balanced multi-trial
binomial-logit, ordinary `latent(..., unique = FALSE)`, fixed q=1, no rank
handoff, no structured/mixed/missing-data surface, and no public inference.
Design 72 supplies the sequential discipline: fixed-rank proof first; only a
passing gate funds the next gate. The initial estimator is a second-order
surrogate with a negative Gaussian KL, not an ELBO, marginal likelihood, REML,
or likelihood-comparable quantity.

## Programme structure

### M1 — release truth and green heavy baseline

- Reconcile all current public behavior, docs, validation rows, and release
  ledgers against the source.
- Repair only independently verified release-blocking defects.
- Re-run targeted, complete, article/pkgdown, source-tarball, exact-SHA Ubuntu,
  three-OS, and heavy checks.
- Close with a validated after-task, check-log, Mission Control update, and
  three fresh D-43 NOT-DONE-default reviews.
- Multinomial 0.6 is verify-only; optional correlation-profile/BCa/structured
  redraw/plotting promotions remain cut or separate.

### M2 — Design 86 EVA scientific admission

- Contract and deterministic proof precede remote compute.
- Totoro is mandatory smoke/reference; one CPU per R process; numerical
  library threads one; <=12 concurrent smoke processes and <=96 operational
  cores, never above the standing 100-core ceiling.
- DRAC-A is authoritative replicated evidence; DRAC-B is parity/overflow when
  available. Never invent an account, module, partition, memory, walltime, or
  throttle. Use `/scratch` for active I/O and `/project` for libraries and
  keepers; never compute on login nodes.
- Freeze source/archive hashes, estimator, comparator, DGP, seeds, runner,
  starts, optimizer controls, thresholds, retry policy, schema, and keeper
  locations before compute.
- Paired Laplace/EVA fits use the identical simulated dataset within a task.
  Every attempt, failure, retry, host, task, diagnostic, gradient, covariance,
  prediction score, runtime, and domain status remains recorded.
- Infrastructure failure permits at most two exact-task retries. Scientific
  failure is evidence and is never retried with new starts/controls/thresholds.
  Source/runner/manifest defects invalidate the entire campaign and mint a new
  campaign ID.
- Stages: C0 deterministic local proof; C1 three-stratum Totoro packet; C2
  DRAC-A/B parity; C3 25 seeds per stratum; C4 100 seeds per promoted stratum;
  C5 separate family/link waves only after q1 scientific GO.
- Outcomes are `SCIENTIFIC-GO` with an explicit allowlist or `CUT` with durable
  negative/partial evidence. No public API is created in M2.

### M3 — public feature admission and source/API freeze

- Candidate surface:
  `gllvmTMB(..., control = gllvmTMBcontrol(integration = "eva"))`.
- Laplace remains default. EVA uses the installed compiled template; no runtime
  temporary DLL.
- Unsupported combinations fail before TMB construction.
- No EVA `logLik`, AIC/BIC, LRT/anova/model weights, `vcov`, confidence/profile/
  Wald intervals, bootstrap/BCa, or ELBO rank selection without separate
  evidence and admission.
- Public metadata names the integration route and unsupported interval status.
- This is the only 0.6 feature-integration window. Freeze source, API,
  allowlist, refusals, feature list, and permissible claims at exit.

## Arc-loop body

At the start of every arc:

1. Read `LOOP/GOAL.md`.
2. Read `LOOP/checkpoint.md` and `LOOP/decision-queue.md`.
3. Select the next dependency-satisfied entry in `LOOP/arcs.md`.
4. If it crosses an open human gate, stop and surface the decision.
5. Otherwise execute with bounded agents and one writer.
6. Verify the actual log and artifacts, not merely exit status.
7. Write/validate the after-task or bounded receipt; commit and push.
8. Update `LOOP/arcs.md`, overwrite `LOOP/checkpoint.md`, and update Mission
   Control.
9. At a batch boundary or heavy context, land the checkpoint and start a fresh
   session from its RESUME line.

At every close, state what the arc did not cover. At programme close,
reconcile plan versus actual and surface all scope/evidence/claim drift.
