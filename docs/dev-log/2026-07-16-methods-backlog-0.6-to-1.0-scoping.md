below):** Wald CI *calibration* for variance components / correlations is a

`sdreport` machinery — no item in this backlog tries to make Wald cover a VC or

## Recommended 0.6 shortlist

**Items 1, 5, then 4, with 2 and 6 as stretch — 3 and 7 deferred to 1.0.**

- **1** is already in-flight and gates the flip decision; it is not optional for
  0.6 to mean anything.
- **5** (unconditional RE redraw) is the best-leveraged item in the whole
  backlog: the fix is a proven, 10-day-old, same-repo pattern
  (`.simulate_eta_unconditional`) that only needs extending to the tiers the
  #18 guard already enumerates, and it is a correctness fix (bootstrap validity)
  rather than an enhancement — it should land before item 4 makes any sense.
- **4** (BCa/studentized) is honestly scoped as a genuine cross-repo gap (no
  code to port), but it is bounded, well-understood statistics, and directly
  extends the bootstrap-repair theme of this session.
- **2** (REML small-n) has an unusually strong in-repo precedent
  (`2026-07-06-blv-honest-interval.md` built the identical mechanism 10 days
  earlier for a different quantity) and targets exactly the n=50 gaussian cell
  the Sigma_unit_diag certificate could not certify — worth a time-boxed
  attempt in 0.6, falling back to 1.0 if the small-n recovery behaviour turns
  out to need its own study.
- **6** (correlation profile) has its design already done (the `Σ_total`
  retarget is specified, not just diagnosed) but is not on the critical path
  for any current certificate — reasonable to defer to 1.0 if 0.6 capacity is
  tight, though the maintainer flagged it 0.6.
- **3** (AGHQ) and **7** (nbinom2 φ-bias) are correctly parked at 1.0: both
  reopen or extend decisions the maintainer already made deliberately (stay
  Laplacian; ship warm-start first), and both are only *partial* fixes even
  when built, so neither should compete with the 0.6 coverage-cert follow-through.
