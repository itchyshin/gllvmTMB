generate the matching favicon set under `pkgdown/favicon/`, and set

## What Did Not Go Smoothly

1. The legacy `make-hex.R` registered its `dev.off()` cleanup via a
   top-level `on.exit({ par(op); dev.off() }, add = TRUE)`. Under
   `source()` in a fresh `Rscript`, the top-level `on.exit` does not
   fire when the script returns. The png device is never closed,
   subsequent `plot.new()` calls fall through to `Rplots.pdf`, and
   no logo is written. Fixed by removing the `on.exit` line and
   adding explicit `par(op); dev.off()` at the end of the script
   in this branch's copy.
2. First commit installed the v6 hex (the script's current design).
   The maintainer pointed out that the legacy production deployment
   used a different design ("the latest, but old"). I fetched the
   actual deployed logo from `itchyshin/gllvmTMB-legacy` via
   `gh api repos/.../git/blobs/<sha>`, swapped it in, and
   regenerated favicons. Cost: one extra CI cycle on PR #9.
3. I did not write this after-task report at the time of the original
   PR #9 commit. The maintainer flagged the gap; AGENTS.md and
   `ROADMAP.md` are explicit about the after-task rule.
   This file is the corrective entry.

## Team Learning

- Prefer **downloading the deployed asset** over regenerating
  from a script when the script's "latest" design is not what
  users see in production.
- Top-level `on.exit` inside an R script is unreliable under
  `source()`; either wrap the script body in a function or call
  cleanup explicitly at the script's end.
- An after-task report is part of the same PR as the work, not
  a separate cleanup pass. This is the AGENTS.md rule and the
  drmTMB habit; both Claude PRs in this session (PR #9 and PR
  #11) skipped it on the first pass and needed this corrective.

## Known Limitations

- `dev/make-hex.R` still produces the v6 design, not the
  deployed v5 (legacy) design that is actually committed to
  `man/figures/logo.png`. If the maintainer wants the script to
  match what ships, the script should be revised or removed in
  a follow-up. Leaving the script as-is preserves the v6 work as
  historical documentation.
