## What Did Not Go Smoothly

Nothing substantive. The rewrite was bounded: opener + reorder
+ one new section. Existing content preserved where possible
(smoke test, tiny example, covariance grid, citation, sister
packages all carried forward verbatim or near-verbatim).

The hardest decisions:

1. **Where to put "Preview status"**: original location was
   between Start here and Install. I folded it into the
   bottom of "What can I model now?" as a one-sentence note,
   so the install instructions come immediately after the
   capability list rather than after a caveat. If the
   maintainer prefers the caveat back as its own section,
   easy revert.
2. **What to do with the "ordination, communality" jargon in
   the original opener**: the new opener does not use those
   terms. Pat reads them later in "Tiny example" ("the
   fitted object reports ordination scores, loadings, Sigma,
   pairwise correlations, and per-trait communality"). The
   abstract concept list is replaced by a concrete fit
   description.
3. **Whether to delete or fold the "data shape is general"
   layouts list**: kept the list (`site x species`,
   `individual x trait`, etc.) but folded it into the
   "What 'stacked-trait' means" section so it has a clear
   anchor sentence ("The data shape is general: ...").

## Known Limitations

- The Start-here URL list still uses absolute URLs
  (`https://itchyshin.github.io/...`). On the rendered
  pkgdown index page this is redundant (the URLs point at
  the same site). Rose audit Section A4 flagged this as
  minor; deferred. A future PR can switch them to relative
  links (`articles/morphometrics.html` etc.) but this affects
  README rendering on GitHub (where relative links may or
  may not resolve).
- The "What 'stacked-trait' means" section is a new top-level
  `##` section. The `_pkgdown.yml` navbar PR (Codex's lane,
  per PR #64 Section I) should not need to reference this
  section by anchor, but if it does, the anchor is
  `#what-stacked-trait-means`.
- "Communality" and "phylogenetic signal" terms still appear
  without inline definition (deferred to the articles). PR
  #64 Section B (B2, B3) flagged these as drift; the index
  page is not the right place to define them, but the
  "Start here" link to `covariance-correlation` does the
  handoff.
- The math display `Sigma = Lambda Lambda^T + diag(s)` now
  appears in "Tiny example" as a follow-up. The opener
  triad's "trait covariance" sentence does not show the
  formula; this is intentional (the opener is for Pat, the
  math is for the next-level reader).
