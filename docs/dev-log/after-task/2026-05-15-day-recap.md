## What did not go smoothly

- **No after-task reports were filed in real time**. The 19 PRs
  after #103 all merged without after-task reports. This file
  catches up retroactively but the per-PR discipline broke.
  Lesson going forward: include the after-task report in each
  PR's same commit, even if it's a short two-paragraph stub.
- **ROADMAP.md drifted by a full day's work** before the
  maintainer flagged it. The 2026-05-14 strategic-plan-revision
  after-task explicitly committed to a "Roadmap tick" line in
  every after-task report; with no after-tasks, no roadmap tick
  fired. Lesson: ROADMAP.md updates should ride in the same
  commit as the work that moves the row, not in a separate
  retroactive catch-up.
- **Cross-reference syntax bit me three times today** (PRs #105,
  #120, #122). Each time R CMD check `--as-cran` with
  `error_on = "warning"` rejected a missing-Rd-link warning.
  Banned patterns: `[fn(args)]`, `[vignette("...")]` (even in
  backticks), `[0, 1]` (interval bounds parsed as autolink
  target), and cross-references to functions on sibling-branch
  PRs. The lesson is now codified in `check-log.md` (point 11,
  added in this PR).
- **WIP discipline slipped**. The soft cap of 3 was exceeded
  multiple times today (peaked at 6 in flight). The user gave
  explicit "keep going" authorisation each time but the result
  was a tangle of cross-PR conflicts on `_pkgdown.yml`,
  `NAMESPACE`, and `NEWS.md` that cost ~5 rebase cycles.
- **Local-only branch pile**. At the time of this report there
  are three uncommitted-to-remote branches:
  `agent/validation-coverage-study` (autolink fix),
  `agent/phase1c-new-simulation-verification` (Concepts
  article), `agent/phase1c-port-corvidae-two-stage` (Worked
  example port). Each one has clean local checks; the push is
  held to manage cross-PR conflict surface. This is a
  reasonable workaround for today's pace but is fragile if a
  rebase corrupts a local branch.

## Going-forward discipline (committed)

Starting with the next PR after this one:

1. **Every PR commit message ends with a `Roadmap tick:` line**
   stating which ROADMAP.md row's status or progress-bar
   changed (or "N/A" if no change).
2. **Every PR ships an after-task report** in
   `docs/dev-log/after-task/YYYY-MM-DD-<short-name>.md` in the
   same commit. Even for trivial fixes, a two-paragraph stub
   beats a missing report.
3. **ROADMAP.md updates ride in the same commit** as the work
   that moves the row, not in a retroactive catch-up PR like
   this one.
4. **WIP soft cap of 3** is taken seriously. When the cap is
   hit, hold new branches locally rather than pushing -- but
   only for short windows (hours, not full days).
