## 1. Goal

The M3.3a smoke (PR #176) and cross-package scout (PR #177)
characterised the under-coverage on count + ordinal + mixed
cells. The Noether audit + Design 48 (this PR) **diagnose the
root cause and prescribe the fix design** without yet writing
the fix.

Two doc artefacts:

1. **Noether identifiability audit** at
   `docs/dev-log/audits/2026-05-18-noether-nbinom2-identifiability.md`
   formally verifies the simulator's `truth$psi[t]` aligns with
   the engine's identifiable `sd_B[t]^2` target (so the
   coverage shortfall is **not** a target-misalignment bug).
   Then shows that $\psi_t$ and $\phi_t$ are **weakly
   identified at small $n$** along a one-dimensional trade-off
   axis — the root cause of the 0.38 nbinom2 coverage.
2. **Design 48** at `docs/design/48-m3-4-boundary-regimes.md`
   prescribes two mitigations: **single-trait warm-start** (per
   gllvm `start.fit=` pattern, Tier A from scout audit) and
   **phi starting-value clamp `[0.01, 100]`** (defensive). The
   implementation goes in a follow-on PR.

**Mathematical contract**: zero engine / API / formula /
extractor / article change. Pure audit + design.

## 3a. Decisions and Rejected Alternatives

> **Decision**: split M3.4 into design (this PR) + implementation
> (follow-on PR).
> **Rationale**: implementation requires careful refactoring of
> `R/fit-multi.R` initial parameter assignment + a new control
> arg + tests + smoke rerun. Bundling with design would make a
> ~600 LOC PR. Same pattern as sparse-Ainv PR #179 (machinery
> first; auto-wiring second).
> **Rejected alternative**: bundle design + implementation.
> Rejected for review-friction.
> **Confidence**: high.

> **Decision**: warm-start default `opt-in` (`control =
> list(init_strategy = "single_trait_warmup")`), not always-on.
> **Rationale**: changes per-fit semantics (different starting
> values may converge to different local modes for ill-
> conditioned likelihoods); v0.2.0 conservative; revisit
> defaulting in v0.3.0 after production R=200 numbers.
> **Rejected alternative**: default-on for count families.
> Rejected for v0.2.0 stability; could revisit.
> **Confidence**: medium (Pat would prefer default-on; safety
> trumps for v0.2.0).

> **Decision**: do NOT include `disp_group=` (shared phi across
> traits) in M3.4 scope.
> **Rationale**: opinionated API choice; needs maintainer
> ratification of the design before implementing; if warm-start
> + phi-clamp suffice for production R=200, disp_group isn't
> needed.
> **Rejected alternative**: include disp_group as Mitigation
> C. Deferred to Design 49 contingency if M3.4 doesn't close
> the gap.
> **Confidence**: high.

> **Decision**: do NOT propose a Bayesian prior on phi to
> regularize the trade-off.
> **Rationale**: scope creep; gllvmTMB is ML-only currently
> (REML post-0.2.0 per README); adding priors is a v0.3.0+
> decision the maintainer should make explicitly.
> **Rejected alternative**: weak prior on log_phi to break the
> (psi, phi) flat-likelihood. Deferred.
> **Confidence**: high.

## 9. Team Learning (per AGENTS.md Standing Review Roles)

**Noether** (audit lead, math): the simulator's `truth$psi[t]`
↔ engine's `sd_B[t]^2` correspondence is exact. The under-
coverage is NOT a target-misalignment bug. Root cause is the
(psi, phi) weak identifiability — the marginal observed-y
variance has two parameters trading off along a one-dimensional
flat-likelihood direction.

**Fisher** (strategy lead): Tier A mitigation is single-trait
warm-start (large impact; addresses root cause). Phi-clamp is
small defensive. Disp_group is medium-impact but opinionated;
defer.

**Boole** (API lead): proposed `control = list(init_strategy =
"single_trait_warmup")` is opt-in for v0.2.0; clean control-arg
surface.

**Gauss** (TMB-side): per-trait `log_phi_nbinom2` is unbounded;
phi-clamp at *starting* values doesn't constrain the optimizer
(which stays free to walk to good values from a sensible
start).

**Curie** (review, tests): before/after smoke comparison is the
right test for the implementation PR; coverage table is the
visible regression metric.

**Pat** (review, opt-in): would prefer default-on for applied
users, but agreed with Boole's v0.2.0 conservative stance —
revisit in v0.3.0 after production R=200 numbers.

**Rose** (review, scope honesty): explicit §4 "what M3.4 will
NOT achieve" prevents the overpromise pattern of 2026-05-15.
The Fisher prediction table in §5 is honest about the band
(~0.70-0.85 expected on nbinom2 at smoke; production gives the
real number).

**Ada** (coordinator): two-PR split (design + implementation)
mirrors sparse-Ainv pattern. Cross-link to Design 49
contingency (disp_group) sets up the next escalation if
needed.
