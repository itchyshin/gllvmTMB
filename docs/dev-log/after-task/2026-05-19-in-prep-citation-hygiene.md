# After Task: in-prep citation hygiene

**Branch**: `codex/in-prep-citation-hygiene`
**Date**: 2026-05-19
**Roles**: Ada (orchestration), Rose (citation hygiene + scope honesty),
Pat (reader-facing clarity), Grace (roxygen/pkgdown verification).

## Goal

Remove or replace "`in prep` / `in preparation`" literature citations in
user-facing help pages and articles when published predecessors exist
or when the citation is not needed for the claim.

## Implemented

- Replaced "`in prep`" citations in roxygen and vignettes with published
  citations already used elsewhere in the package (e.g. Westneat,
  Wright & Dingemanse 2015; Nakagawa & Schielzeth 2010; Nakagawa,
  Johnson & Schielzeth 2017) or removed the citation when it was not
  essential to the statement.
- Aligned README's package citation with `inst/CITATION` (software
  manual form) instead of citing an unpublished methods paper.
- Updated README references to `meta_V(scale = "proportional")` to cite
  Nakagawa (2022) rather than an in-prep placeholder.

## Mathematical Contract

No model, likelihood, formula grammar, estimator, or public API changed.
Edits are documentation-only (roxygen text, README prose, vignette
prose).

## Tests Of The Tests

This slice changes documentation only. The guardrail is a mechanical
pre/post sweep for `in prep|in preparation` plus roxygen and pkgdown
checks that would fail if documentation became inconsistent.

## Consistency Audit

- README citation wording now matches `inst/CITATION`'s software-manual
  entry (year + version + URL) rather than implying a completed methods
  paper.
- Roxygen help topics regenerated and `pkgdown::check_pkgdown()` stayed
  green, confirming reference topics remain discoverable.

## Known Limitations

- This slice does not attempt a full reference-style unification across
  all articles (punctuation, italics, DOI format). It only removes the
  in-prep placeholders.
- This slice does not resolve the broader "foundations to cite" debate
  for each modelling component; it uses only citations already present
  in the repository.
