# After-task report: R bridge fit-time no-X CI admission

Date: 2026-06-16

Branch: `codex/r-bridge-grouped-dispersion`

## Scope

Admitted a narrow fit-time confidence-interval route for ordinary
`gllvmTMB(..., engine = "julia")` fits. The public `gllvmTMB()` signature now
accepts `ci_method`, `ci_level`, `ci_nboot`, and `ci_seed` and forwards those
controls to the existing Julia bridge CI payload path for no-X Gaussian,
Poisson, and Bernoulli binomial rows.

Still gated: grouped-dispersion CI endpoints, per-trait ordinal CI endpoints,
masked CIs, mixed-family CIs, X-row CIs, non-Gaussian REML CIs, structured
terms, `newdata` prediction/simulation, unconditional random-effect redraws,
and broad/native CI parity.

## Mathematical contract

No likelihood, formula grammar, family, parameterisation, or TMB code changed.
This slice only routes the already-admitted bridge CI options:

```text
ci_method in {"none", "wald", "profile", "bootstrap"}
ci_level  in (0, 1)
ci_nboot  integer bootstrap replicate count
ci_seed   integer bootstrap seed
```

For native `engine = "tmb"` fits, non-default fit-time `ci_*` arguments stop
and tell the user to fit first and use `confint()` with the desired method.

## Team learning

Ada: The clean public surface was `ci_*` on `gllvmTMB()` because these are
inferential payload choices already present in the Julia bridge, not Julia
solver choices.

Hopper: The bridge route needed to work both for direct `gllvm_julia_fit()` and
ordinary `gllvmTMB(..., engine = "julia")` calls, while retaining the existing
post-fit `confint()` recomputation path.

Fisher: Fit-time CI support is not equivalent to broad inference support. The
scope stays no-X Gaussian, Poisson, and Bernoulli binomial; grouped-dispersion,
ordinal, masked, mixed-family, and X-row intervals still require separate
endpoint/status rows.

Boole: `engine_control` should stay reserved for future Julia-side algorithm,
solver, tolerance, and threading choices. Reusing it for CI methods would blur
the R/Julia and DRM/GLLVM argument contract.

Rose: The stale wording to remove was “no fit-time Julia CI control surface.”
The replacement wording has to name both the admitted rows and the still-gated
CI regimes.

Shannon: The pre-edit census found no open PR collision and only current local
programme commits on hot files. The after-task/report/check-log pairing is
present for the slice.
