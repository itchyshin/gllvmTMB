## 1. Goal

Walk `MIS-07` (`predict.gllvmTMB_multi()` link / response prediction) from stale
partial wording to a covered, evidence-backed validation-register row.

## 2. Implemented

Added a focused mixed-family prediction test for Gaussian / binomial / Poisson
fits. The test verifies that training-row link predictions match
`fit$report$eta`, response predictions use each row's trait/family inverse
link, and `newdata` response predictions map rows back to the training trait's
link before transformation.

Updated `predict.gllvmTMB_multi()` roxygen and
`man/predict.gllvmTMB_multi.Rd` to describe link / response scale output without
overclaiming. Updated the extractor contract so `extract_*()` remains
link-scale while response-scale fitted values belong to
`predict(type = "response")`. Promoted `MIS-07` to `covered` and recounted the
register as `172/22/0/7` over 201 rows.

## 6. Tests of the Tests

The new test would fail if `predict(type = "response")` applied the first
family's identity link to every row, because binomial rows would no longer be
bounded in `(0, 1)` and Poisson rows would not match `exp(eta)`. It also checks
that `newdata` response predictions use the trait-specific link, not a single
global inverse link.

The existing robustfix test in `test-missing-data-robustfix.R` remains an
independent regression guard for the same first-family inverse-link failure
mode on mixed-family fits.
