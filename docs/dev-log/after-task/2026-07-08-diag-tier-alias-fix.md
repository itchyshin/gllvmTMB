# After-task — `diag` tier-claiming fix (aliased species-level diagonals)

**Date:** 2026-07-08 · **Agent:** Claude (Ada) · **Branch:** `claude/blv-coverage-breadth`
**Status:** fix landed on branch + tested; **not merged** (high-risk: likelihood/spec change).

---

## 1. Goal

Begin the `B_lv` calibrated-interval breadth arc. Slice **S1** (composability probe across
`phylo` / `animal` / `kernel` / `spatial`) surfaced `pdHess = FALSE` on *all four* tiers,
including the known-good phylo control. Diagnosing that became the task: find the true cause,
and fix it if it is a defect rather than a property of the model.

## 2. Implemented

**One-line source change** (`R/fit-multi.R`): a per-trait `diag` covstruct is now claimed by
**exactly one** tier. Previously, when `unit == cluster`, a single `diag` covstruct on that
grouping satisfied **both** tier predicates and materialised **both** engine slots:

- unit tier — `use_diag_B` / `s_B`, scaled by `sd_B` (`src/gllvmTMB.cpp:1821`, indexed by `site_id`)
- cluster tier — `use_diag_species` / `q_sp`, scaled by `sd_q` (`src/gllvmTMB.cpp:1839`, indexed by `species_id`)

When the two index vectors coincide, `s_B(t,g) + q_sp(t,g)` collapses to one
`N(0, sd_B² + sd_q²)` draw. **Only the sum is identified.**

Consequences, all verified:

- The Hessian gains exactly `n_traits` flat directions, supported **50/50 on the two diagonal
  blocks and on nothing else** (`theta_rr_B`, `theta_rr_phy`, `alpha_lv_B` carry **zero** weight).
- `pdHess = FALSE` ⇒ `sdreport` refuses ⇒ **every Wald SE silently becomes `NA`**.
- The split is **arbitrary**: walking it from 50/50 to 95/5 at fixed sum moves the objective by
  `~1e-9`. It lands on `0.500` only because the optimiser starts symmetric.
- `extract_Sigma(level = "unit", part = "unique")` returned **exactly half** the identified `Psi`.

Also added `tests/testthat/test-diag-tier-alias.R` (5 tests, 14 assertions).

**Second change (Defect 2, maintainer-authorised in the same session).** `H²`'s denominator `V_eta`
is now explicitly the **species-level** latent variance: only components whose grouping is the
`cluster` column enter it. A single shared rule, `.phylo_signal_unit_is_cluster(fit)`, drives both
`extract_phylo_signal()` (`R/extract-omega.R`) and the CI kernel
`.phylo_signal_H2_from_report()` (`R/phylo-signal-ci.R`), which previously disagreed. The new
argument `unit_is_cluster` is **mandatory, with no default** — a silent default is exactly how this
class of bug arises. Roxygen now documents the choice *and* the rejected alternatives.
Added `tests/testthat/test-phylo-signal-species-level.R` (3 tests).

## 3a. Decisions and Rejected Alternatives

- **Chosen:** index-wise partition — exclude from the cluster predicate any covstruct already
  claimed by the unit tier. Crossed `site × species` designs (the `q_it` term of the Nakagawa
  et al. functional-biogeography framework) are untouched by construction, because a
  cluster-grouped covstruct is never claimed by the unit tier.
- **Rejected — map one block off downstream** (at `fit-multi.R:3668`, where the machinery
  already exists). It would fix `pdHess` but leave `fit$use$diag_species` lying, and the
  extractors branch on that flag.
- **Rejected — fail-loud guard instead of a fix.** Considered (maintainer offered it). Rejected
  because the configuration is the package's *default* `latent(unique = TRUE)` at `unit == species`
  — refusing it would break the flagship phylogenetic path.
- **Rejected — make `unit == cluster` an error.** Same reason.
- **Not attempted — repairing the downstream extractors.** They read one tier. Post-fix the
  `unit == species` case is correct automatically (one tier holds the whole `Psi`); the crossed
  case exposes a *definitional* disagreement (see §7a Defect 2) that needs a maintainer ruling,
  not a patch.

## 6. Tests of the Tests

- The new test file was written **first** and run against unmodified `main`: **8 failures**, each
  on the intended assertion (`use$diag_species`, `theta_diag_species` present in `opt$par`,
  `sd_q` reported, 3 flat eigenvalues, `pdHess FALSE`, Wald `std.error` non-finite).
- The **crossed-design regression guard passed before *and* after**, which is what proves the fix
  is targeted rather than a blanket removal of the cluster tier.
- One assertion was **withdrawn as fragile**: an absolute `Psi > 0.17` threshold calibrated on a
  different cell (`S=40, T=5`) would not have held at the test's `S=30, T=4`. Replaced with the
  behavioural identity `psi == exp(2 * theta_diag_B)` plus `extract_Sigma(level = "cluster")`
  must abort.
- Ridge walk (`obj$fn` at fixed variance sum, share `0.50 → 0.95`, Δ `~1e-9`; `B_lv` Δ exactly
  `0.00e+00`) is the independent, optimiser-free confirmation.
- **Defect 2's decisive test is a cross-check, not a value assertion.** `test-phylo-signal-species-level.R`
  asserts that the **two independent `H²` code paths agree** on the same fit. Value assertions
  (`H² < 1`, `Psi > 0`) would have caught the symptom; the agreement test catches the *class* — a
  second implementation drifting from the first. That is exactly what happened here, and the header
  comment at `phylo-signal-ci.R:76-79` had already documented the intended rule while the code below
  it did something else.

## 7a. Issue Ledger

- **Defect 1 — FIXED on branch.** Aliased `diag_B` / `diag_species` at `unit == cluster`.
- **Defect 2 — FOUND, VERIFIED, and FIXED** (maintainer ruling 2026-07-08: keep the **species-level**
  denominator `σ²_phy + q_it`, and document that other definitions exist).
  `extract_phylo_signal()` (`R/extract-omega.R:409-414`)
  builds `V_eta` from `extract_Sigma(level = "unit", part = "unique")`, i.e. `sd_B²` alone. Its own
  comment — *"Psi_non = Psi_B (the species-level diag term)"* — assumes only one diagonal exists.
  In the **intended crossed design** (`unit = "site"`, `cluster = "species"`,
  `phylo_unique(species) + unique(0 + trait | species)`) `use$diag_B` is FALSE, so `Psi_diag = 0`
  and **`H² = 1.00` is reported for every trait.**
  Reproduced with the package's own fixture DGP (`sigma2_phy = 1.5`, `q_it = 0.7`, true
  `H² = 0.682`): `extract_phylo_signal()` returns `H2 = 1, C2_non = 0, Psi = 0` for all 4 traits,
  while `fit$report$sd_q = 0.64 / 0.43 / 0.87 / 0.45` is nonzero. It even emits
  `"Psi_non = 0 across all traits"` and reports 100 % signal anyway.
  **This survives the Defect 1 fix.** Worse, the two `H²` paths disagreed on the *definition*:
  `extract_phylo_signal()` → `sd_B²`; `phylo-signal-ci.R:111-119` → `sd_B² + sd_q²`; the arguably
  correct species-level denominator is `σ²_phy + q_it` — **none of the three agreed.**
  **Maintainer ruling (2026-07-08): species-level, `σ²_phy + q_it`, with the alternatives
  documented.** Implemented: `V_eta` now admits only components whose grouping is the `cluster`
  column. Unit-tier `Lambda_B` / `sd_B` enter **iff** `unit == cluster`; otherwise the cluster-tier
  `q_it` (`sd_q²`) does. Both `H²` code paths now share one rule (`.phylo_signal_unit_is_cluster()`)
  and agree to `0.00e+00` on the same fit. The crossed fixture now returns
  `H² = 0.62 / 0.86 / 0.65 / 0.81` (truth `0.682`) instead of `1.00 / 1.00 / 1.00 / 1.00`.
  Note the header comment at `phylo-signal-ci.R:76-79` already said *"`sd_B²` **or** `sd_q²`"* while
  the code summed both — the intent was always "exactly one"; Defect 1's tier-claiming makes that
  true by construction.
  The `Psi_non = 0` advisory also told crossed-design users the wrong remedy (it suggested a
  unit-tier `latent()`); it now points at `indep(0 + trait | <cluster>)`.

## 8. Consistency Audit

Swept the neighbourhood for the same class (single-tier reads of an aliased pair) via a scoped
sub-agent over `R/` + `src/gllvmTMB.cpp`. Result: **12 consumers** read one block alone; only
`extract_Sigma(part = "shared")` (loadings only) and `phylo-signal-ci.R`'s `H²` (which sums both)
are safe. Full table in Defect 2's task entry.

Checked the blast radius of the fix: **43 test files** use `unit = "species"`, but the fix only
fires where a `diag` covstruct *also* sits on that grouping. Both `q_it` tests
(`test-phylo-q-decomposition.R`, `test-phylo-signal-ci.R`) use the crossed design and are
unaffected — verified by reading their `unit=` / `cluster=` arguments, not by assumption.

**The bug was already known.** `tests/testthat/test-phylo-signal-ci.R:12-17` states it exactly:
*"Avoids the over-parameterisation seen when unit == species (both diag_B and diag_species fire
from a single `unique` term)."* It was diagnosed, written into a test comment, routed around in a
fixture — and never fixed at source. Then `latent(unique = TRUE)` became the default, which makes
it fire on every Model A fit.

## 9. What Did Not Go Smoothly

- **The installed package was stale** (built 2026-07-06 06:45; commit `401d0763`, which admits REML
  for Gaussian `lv`, landed after). The first probe failed with `"lv does not support REML = TRUE"`
  — a *package-staleness* artifact that reads exactly like a capability gap. Rebuild before probing.
- **`obj$he()` errors** on random-effects models (`"Hessian not yet implemented"`). Use
  `optimHess(par, obj$fn, obj$gr)`.
- **Two of my own hypotheses were wrong and had to be discarded on evidence:**
  (a) "rotation indeterminacy" — ruled out *a priori*: at `d = 1` the only symmetry is a discrete
  sign flip, which cannot make a Hessian singular. (b) "single-replicate data confound" — refuted
  by adding replicates, which did **not** restore `pdHess`.
- **A jitter test was inconclusive and I nearly read it as evidence.** `n_init`/`init_jitter`
  produced byte-identical `logLik`, i.e. the control never perturbed anything. Replaced with the
  optimiser-free ridge walk.
- ⚠️ **`devtools::test()` DELETED 11 committed snapshot files.** Because the visual-snapshot tests
  were *skipped* (not run), `testthat` judged their `_snaps/plot-visual-snapshots/*.svg` "unused"
  and removed them. Restored with `git checkout --`. **Anyone running `devtools::test()` in this
  repo must check `git status` afterwards.** Worth a `.Rbuildignore`/CI note or
  `testthat::snapshot_accept` discipline — a skipped test should never delete its own baseline.

## 11. Team Learning

1. **A workaround comment is not a fix.** The defect sat, correctly diagnosed, inside a test-fixture
   comment while the default code path drove straight into it. When you route a fixture *around* a
   bug, open an issue in the same commit — or the next person inherits a landmine with a sign on it
   that only they can't see.
2. **Never let a symptom acquire a causal story without checking the null space.** `pdHess = FALSE`
   got attributed in a design doc to a plausible-sounding variance trade-off. Four eigenvectors
   said otherwise in about a minute. That wrong story then *rationalised deleting the Wald arm* from
   the method set — a wrong diagnosis is not inert; it propagates into design.
3. **Test the flat direction, not the flag.** `pdHess` is a second-order boolean. The decisive
   evidence was walking the ridge and watching the objective (Δ `~1e-9`) and `B_lv` (Δ exactly `0`).
   This is the hub's "trust recovery-to-truth over second-order flags" guard, paying out again.
4. **Rebuild before you probe.** A stale install fabricated a capability gap that did not exist.
5. **Two implementations of one formula will drift; test that they agree, not that they are right.**
   `H²` had two implementations. One read `sd_B²`, one read `sd_B² + sd_q²`, and a *third* rule was
   written in a comment above one of them. A value assertion on either would have looked fine in the
   fixture that happened to make them coincide. The test that catches this class is
   `expect_equal(path_a(fit), path_b(fit))`.
6. **A mandatory argument beats a sensible default.** `.phylo_signal_H2_from_report()` gained
   `unit_is_cluster` with **no default**. A default of `TRUE` would have been "backwards compatible"
   and would have silently reproduced the bug at any call site the next person forgets to update.
7. **Don't reinstall a package while its tests are running.** I did, mid-heavy-run, and had to
   discard the results and re-run. The R process holds the old `.so`; the outcome is neither the old
   build nor the new one.
