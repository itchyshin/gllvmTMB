Three modes: indep = per-trait diagonal · dep = full unstructured ΣT (T(T+1)/2 params) · latent = ΛΛT with optional diagonal Ψ companion. Ordinary latent() carries Ψ by default; source/kernel latent terms are loadings-only unless unique = TRUE. The scalar (one shared trait variance) case is the modifier common = TRUE on any indep term — e.g. indep(0 + trait | g, common = TRUE), intercept-only. The scalar()/*_scalar() keywords are soft-deprecated compatibility syntax for it.

Where the gaps are · 0.5.0 → 1.0

  • Interval coverage — the headline gap. Zero families have coverage-checked intervals; every cell is point-only. Calibration is the largest 1.0 task.
  • Random slopes (≥1) are unfinished. In development / diagnostic-grade even for Gaussian — not a finished 0.5.0 capability. Shown as partial, not ✓.
  • Missing responses: core families now tested. Gaussian + poisson/NB2/binomial masked-response is verified (sentinel-invariance, 2026-07-12); nbinom1 recovery + mi() predictor coverage pending.
  • Fitted diagnostics reach 3 families (Gaussian, Poisson, NB2). Extending residual / predictive checks to more families is the next diagnostics step.
  • Categorical needs major development — nominal support is early-stage, not a usable path yet.
  • REML is a Gaussian-only pilot. Non-Gaussian, weighted, and missing-data REML are later work.
  • Delta / hurdle latent-scale correlation awaits a reporting convention (two response scales) — a design gap, not a bug.
  • Grammar symmetry: the covariance grid is three modes {indep, dep, latent}; the one-shared-variance ("scalar") case folded into the common = TRUE modifier on any indep term (the scalar()/*_scalar() keywords are soft-deprecated compatibility syntax). The correlated-vs-uncorrelated (| / ||) random-slope axis is now built: mode(1 + x || g) = intercept ⟂ slope (indep|| fully diagonal, dep|| = Σint⊕Σslope via a Cholesky parity pin, latent|| the separate-Λ form) across phylo / animal / kernel / spatial + source-tier latent, recovery/target-matrix tested (Design 79 §4).

Reconciled against NAMESPACE family exports, NEWS.md (diagnostics scoped to Gaussian/Poisson/NB2; single-trial Bernoulli diagonal RE mapped off; Student-t variance NA for df ≤ 2), the keyword grid, and the README support boundary, plus maintainer-confirmed status for the in-progress lanes (random slopes ≥1 unfinished; missing responses validated for Gaussian only; categorical early-stage). Where per-family structured or slope support is not individually validated in code, the column reflects grammar availability, not a coverage claim — firming these into per-cell evidence is a 1.0 audit, not a 0.5.0 promise.

Estimation methods — the calibration ladder

Why extraction is point-only: the tools that would certify a calibrated variance component (Design 80's "Bar 3") are not all in the engine yet.

MethodGaussian (exact likelihood)Non-Gaussian (Laplace)
ML✓ done default✓ done default; RE SD biased down at small n

The ladder follows docs/design/80-nongaussian-re-evidence-bars.md. A large ML-vs-REML gap is the cheap diagnostic that flags the small-cluster regime where the variance estimate is untrustworthy — the natural hook for the AGHQ work.
